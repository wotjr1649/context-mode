# ctxscribe — Repo/Plugin Rename Design Spec

- **Date**: 2026-07-13
- **Status**: Design (v3 — post 2 subagent review rounds) — awaiting user OK before implementation plan
- **Type**: Identity migration (fork-scale, both Claude Code + Codex clients)
- **Review history**: 2 rounds × 2 independent subagents. Round 1 doubled the known surface
  (Claude key-matchers, full Codex identity, build gate). Round 2 found more stragglers
  (Codex hook commands, version-drift heal regexes, extra funnels) + a §1 contradiction — which
  proved that **hand-enumerating the surface is inherently leaky**. v3 therefore pivots §4 to a
  **rule + exclusion-set + post-sweep assert-grep** model so completeness is by-construction,
  and keeps a hand-list only for **non-mechanical edge cases**.

## 1. Goal

Re-identify the ELv2 hard fork from plugin `context-mode` / marketplace `context-mode-js` to an
independent project identity **`ctxscribe`**, in preparation for developing it as the user's own
product. Keep the codebase, the ELv2 license + upstream attribution, and the user-facing `ctx_*`
tool surface. **Rename, NOT rewrite, NOT relicense.** "scribe" = records/curates the FTS5
knowledge base + session memory.

Identity spans **two clients** (both charter-supported): Claude Code and Codex. The ONLY
`context-mode`-bearing things intentionally kept are the **`CONTEXT_MODE_*` env-var names**
(§4b) and the **ELv2 attribution + legacy `~/.context-mode` dot-dir** (§4b) — everything else
functional is renamed (incl. the active on-disk data dir → `ctxscribe/`, §4d).

## 4. Rename surface — RULE-BASED (do not hand-enumerate; verify by grep)

### 4a. Transform rules (apply everywhere EXCEPT §4b invariants)

| identity token | → |
|---|---|
| `context-mode@context-mode-js`, `context-mode@context-mode` (plugin KEY, both clients) | `ctxscribe@wotjr1649` |
| marketplace `context-mode-js` / Codex marketplace `context-mode` | `wotjr1649` |
| plugin/pkg name `context-mode`, npm `bin`/`command` `context-mode`, `context-mode hook …` | `ctxscribe` |
| MCP prefix `plugin_context-mode_context-mode` | `plugin_ctxscribe_mcp` |
| `mcpServers`/`mcp_servers` key `context-mode` | `mcp` |
| cache-path literals `…/context-mode-js/context-mode/…`, `…/context-mode/context-mode/…` | `…/wotjr1649/ctxscribe/…` |
| repo `wotjr1649/context-mode` | `wotjr1649/ctxscribe` |
| skill ns `context-mode:` | `ctxscribe:` |
| cache-heal hook file `context-mode-cache-heal.mjs` | `ctxscribe-cache-heal.mjs` |
| active data-dir segment `<configDir>/context-mode/` | `<configDir>/ctxscribe/` (§4d) |

### 4b. Exclusion / invariants (rule must NOT touch these — a naive `s///g` breaks the fork)

- **ELv2 / D9 attribution**: `docs/UPSTREAM-CREDITS.md`; any `mksglu/context-mode`;
  `registry.npmjs.org/context-mode`; `assert-no-upstream-mksglu.mjs` needles;
  `.claude-plugin/plugin.json` `author.url` (`github.com/mksglu`); the `context-mode-ops` skill
  name in credits.
- **`CONTEXT_MODE_*` env-var names** (~274 occ in tests + runtime) — KEEP (§9 D2). **TRAP**:
  `hooks/core/routing.mjs:576` const is *named* `CONTEXT_MODE_SUBSTRING` (matches the KEEP
  pattern) but its **value `"context-mode"` MUST change** — see §4c.
- **Legacy home dot-dir `~/.context-mode`** (`server.ts:628/4355` stale-content cleanup +
  `platform-bridge.mjs:38` fallback) — a genuinely older pre-isolation location; no `.ctxscribe`
  legacy exists → KEEP.
- Path-derived heal core (§3.3 auto-adapting list); `ctx_*` tool names; `ctx-*` skill sub-names.

### 4c. Non-mechanical sites — need special handling (NOT a plain string swap)

1. **npm-funnel cleanup** (charter-adjacent): `src/cli.ts:644-645`, `src/util/plugin-cache-integrity.ts:150`,
   `scripts/plugin-cache-integrity.mjs:244` (+ check `src/server.ts` upgrade strings) emit
   `npm install -g context-mode` / `bun add -g context-mode`. Fork is `private` → do NOT flip to
   `ctxscribe` (nonexistent npm pkg); rewrite to the **marketplace-install path** instead.
2. **Routing own-tool detection** (definite change, not "verify"): `hooks/core/routing.mjs:576/585`
   — after the prefix change, non-`execute` `ctx_*` tools (`ctx_search`, `ctx_index`,
   `ctx_fetch_and_index`, `ctx_stats`, …) mis-classify as external MCP (only `ctx_execute/_file/
   batch` have branches). Key off the unchanged `ctx_` leaf; update `tests/hooks/core-routing.test.ts`
   in lockstep. (This is the `CONTEXT_MODE_SUBSTRING` value-vs-name trap from §4b.)
3. **Data-dir override/default asymmetry**: `src/adapters/base.ts:120-121`, `codex/index.ts:480-481`
   — the OVERRIDE path carries the `context-mode` segment, the DEFAULT path does not. Flip both
   consistently or one install writes `sessions/` under `ctxscribe/` but `memory/` under
   `context-mode/`. Plus debug-log literals `hooks/precompact.mjs:31`, `hooks/codex/precompact.mjs:27`,
   `heal-partial-install.mjs:426`, and `db-base.ts:417 defaultDBPath(prefix)` ↔ `store.ts:185/423`
   tmp-DB cleanup regex `/^context-mode-\d+\.db$/` (keep lockstep).
4. **Codex hook launch commands** (D4): `configs/codex/hooks.json:7/14/21/28/35/42` +
   `src/adapters/codex/index.ts:102-107` (`HOOK_MAP`, must match the committed template;
   asserted by `tests/core/cli.test.ts:3062`) → `ctxscribe hook codex <event>`.
5. **Sentinel couplings** (rename both sides together): `src/session/extract.ts:111`
   (`startsWith("context-mode:")`) ↔ `hooks/core/formatters.mjs:163` (emits `"context-mode: …"`);
   `hooks/core/routing.mjs` redirect markers ↔ `session-loaders.mjs`/`retrieval-marker.ts`.
6. **Command-matchers** (D4): `src/adapters/claude-code/hooks.ts:212`, `claude-code/index.ts:530`
   (`includes("context-mode hook")`); `scripts/postinstall.mjs:170/175/182/210/215/267`
   (bin/pkg/upgrade-dir — only runs on real `npm install -g`, but flip for consistency).
7. **MCP self-name** `src/server.ts:146` (`name:"context-mode"`) — near-cosmetic (client keys off
   the `mcp` server key), flip for consistency.

### 4d. Active data dir (§9 D1)

Rename the `<configDir>/context-mode/{sessions,memory,content}/` literal to `ctxscribe/` at ALL
sites (§4c #3 covers the asymmetry; grep the literal to enumerate — ~13 files, do not trust a
fixed count). **No auto-migration code**: user renames the physical dir or starts fresh. Old
data orphans safely (confirmed: no half-migrated reader; fresh SQLite at the new path; no lock/WAL
contention). Legacy `~/.context-mode` stays (§4b).

### 4e. Tests

`tests/` hardcodes the old identity: **~1049 `context-mode` occurrences across 93 files** (must
rename) **plus ~274 `CONTEXT_MODE_` occurrences across 38 files (must PRESERVE)**. Highest-risk:
`heal-installed-plugins.test.ts` (153), `cli.test.ts` (90), `codex.test.ts` (89),
`shell-snapshot-heal.test.ts` (64, anti-spoof), `tool-naming.test.ts` (exact-prefix),
`core-routing.test.ts` (the §4c #2 change), `asymmetric-drift-assert.test.ts` (build-gate sibling,
reads `mcpServers["context-mode"]`), `cache-heal-self-heal.test.ts` (hook-name),
`cli-fork-origin.test.ts` (every URL `wotjr1649/context-mode` → `ctxscribe`; KEEP the `mksglu`
reject case). Same exclusion set as §4b.

### 4f. Completeness gate (the guarantee — replaces hand-enumeration)

A scripted `assert-identity-clean` (sibling of `assert-no-upstream-mksglu`, wired into
`npm run build` + phase 4): grep runtime source + tests for `context-mode`/`context-mode-js`
identity tokens, **whitelisting** the §4b invariants (`CONTEXT_MODE_[A-Z]`, `mksglu/`,
`registry.npmjs.org/context-mode`, `context-mode-ops`, `~/.context-mode`/`.context-mode` dot-dir,
UPSTREAM-CREDITS, `author.url`). Must return zero. This is what makes "no functional ref remains"
verifiable rather than enumeration-dependent.

## 5. Phasing

1. **Claude functional** (rules §4a + edge cases §4c that are Claude-side): manifests, KEY +
   matchers (§3.3 list), MCP prefix, cache-path fallbacks + version-drift regexes, repo/
   isForkOrigin, server key, cache-heal hook + generator + old-hook cleanup, build gate, routing,
   funnels, fallback literal.
2. **Codex functional** (§4a + §4c #4): manifests, `.agents/plugins/marketplace.json`,
   `codex/index.ts` (key/server/HOOK_MAP), `.codex-plugin/mcp.json`, `.mcp.json.codex.example`,
   `configs/codex/{config.toml,hooks.json}`.
3. **Data dir** (§4d) + **cosmetic sweep** (comments/logs/README/CONTRIBUTING/CLAUDE.md — the
   obsolete collision workaround) + **tests** (§4e), all under the §4b exclusion set.
4. **Build + gates**: `typecheck`, `npm run build` (bundle drift 0), `assert-bundle`,
   `assert-asymmetric-drift`, `assert-no-upstream-mksglu`, **`assert-identity-clean` (new, §4f)**,
   capped `vitest` (baseline 6 known failures; NO new failures).
5. **Version** → 1.0.0, `npm run version-sync` (no `v1.0.0` git tag — exists; §9).
6. **Migration/deploy** (approval-gated, destructive-remote): GitHub repo rename + `origin`
   (do not sync old marketplace mid-cutover); `marketplace add wotjr1649` + install
   `ctxscribe@wotjr1649` (+ Codex); `verify-deploy.mjs 1.0.0` PASS; remove old install +
   marketplace; run N1 orphan-hook cleanup; reload; live `ctx_*` smoke (Claude new prefix + Codex
   bare).

## 6. Risks & mitigations

- **Migration not in-place** (new key → new cache path) → fresh install + old removal; keep old
  until new `verify-deploy` passes.
- **Dual-install overlap** → two MCP servers boot: two `ctx_*` toolsets (different prefixes) and
  **two SEPARATE KBs** (old `…/context-mode/content/*.db`, new `…/ctxscribe/content/*.db` — D1
  gives separate dirs, so NOT a shared KB; session/memory splits across two DBs during the
  window, benign, resolves on old removal), two `pretooluse.mjs` catch-alls (double-fire). Heal is
  key-scoped so neither reaps the other — provided §3.3's matchers (incl. `orphan-reaper.mjs:117`)
  are all updated. Minimize the window.
- **Orphaned cache-heal hook (N1)** → old `~/.claude/hooks/context-mode-cache-heal.mjs` + its
  settings entry persist; new start.mjs must unlink + de-register (mirror the `.sh` cleanup).
- **GitHub-rename window** → 301 makes the old marketplace resolve to new code; don't
  `marketplace update` the old one before removal.
- **Version-reset tag collision** → `v1.0.0`–`v1.0.3` exist; do NOT recreate; deploy uses
  `version` + marketplace, not tags (§9).
- **Partial data-dir flip** (§4c #3) → override-vs-default asymmetry; enumerate + assert-grep.
- **Re-touching stabilized machinery** → same rigor (independent review + live verify); `src/`
  edits → `npm run build`; pre-commit enforces drift 0.

## 7. Acceptance criteria

- `typecheck` clean; bundle drift 0; `assert-bundle` + `assert-asymmetric-drift` +
  `assert-no-upstream-mksglu` + **`assert-identity-clean` (§4f)** all pass.
- Capped `vitest` == baseline (no NEW failures) — requires the `tests/` rename (§4e).
- `verify-deploy.mjs 1.0.0` PASS on `ctxscribe@wotjr1649`; deployed tree byte-identical to commit.
- Live `ctx_*` smoke under `mcp__plugin_ctxscribe_mcp__*` (Claude) + bare `ctx_*` (Codex).

## 8. Out of scope

- Rewrite / clean-room; relicense (stays ELv2); npm publish (stays `private`); new features;
  `ctx_*` tool rename; **auto**-migration of the data dir (literal IS renamed, §9 D1; only the
  on-boot move is skipped); `CONTEXT_MODE_*` env rename (§9 D2); legacy `~/.context-mode` rename.
