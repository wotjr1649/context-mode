§ 01 · The plugin

For developers

Free, open-source, ELv2. Runs locally on your machine. No cloud, no telemetry, no account.

context-mode keeps raw data out of the LLM context.

Your AI coding agent burns through context. Every gh issue list, every grep, every file read piles into the conversation and gets re-sent every turn. Fifty turns in, that one command has cost you 750,000 input tokens.

context-mode is an MCP plugin that intercepts large tool output before it touches context. Raw data goes into a local FTS5 store. The agent searches it when it needs to. Same answers. Same work. 30× fewer tokens.

Works on 17 adapters — Claude Code, Cursor, Copilot, Codex, Gemini CLI, JetBrains Copilot, GitHub Copilot CLI, Antigravity CLI, Kiro, and 8 more. 331,200+ developers run it locally today.

§ 02 · The Platform

For engineering orgs

$20 / seat / month. Same plugin, opt-in event forwarding. Privacy-first.

You're already saving money. Reinvest a fraction of it here.

Your team is already running context-mode on every machine. Every session saves up to 98% of context per call — real budget back in your pocket, every day, on every developer. The OSS plugin pays for itself before lunch.

Context Mode Platform takes a small slice of those savings and turns the plugin's existing data into an org view your CTO, EM, and CISO can defend. You're not adding a new tool — you're activating a layer that's already there.

How the data flows

The OSS plugin already captures structural events on every tool call — tool name, file path, error counts, decisions made. Locally, in a SQLite store on the developer's machine. This is happening right now, for free, whether you're a Platform customer or not.

When you opt into Platform, those same events forward to your private org workspace over an authenticated channel. Never source code. Never prompt content. Never file content. Just the structural metadata your engineers' workflow already generates. One switch, no new install, no developer interruption.
